Prerequisites

What you need before first launch.

The desktop app is a developer-focused build. It expects Python and ffmpeg on your machine, plus your own API keys. We’ll bundle these in a future release.

Python 3.12

brew install python@3.12. The app launches a local Python backend and expects it on your PATH.

ffmpeg

brew install ffmpeg. Used to enhance and chunk audio tracks for transcription.

Your own API keys

An OpenAI key for recap drafting, an AssemblyAI key for transcription. Both stay on your machine. We never see them.

Install

Three clicks to your first transcript.

01

Drag to Applications, right-click Open

Open the .dmg, drag Videoscriber into Applications. First launch only: right-click the app and choose Open. Gatekeeper blocks double-click on unsigned apps. One-time prompt; Apple code-signing is on the roadmap.

02

Wait one minute

On first launch the app sets up its local Python environment and installs dependencies automatically (~1 min). Nothing for you to do. Just wait for the window.

03

Paste your API keys

The app opens on a setup screen. Paste your OpenAI and AssemblyAI keys (both free to sign up), hit Save keys, then quit and reopen Videoscriber once. You’re ready to drop a recording.

{# ================================================ DATA STAYS ON YOUR MAC === #}
Your data, your machine

Nothing ever leaves, unless you send it.

The desktop build talks directly to the providers you configure. No Videoscriber server in the middle, no account, no telemetry.

Local SQLite database

Transcripts, recaps, and metadata live in ~/Library/Application Support/Videoscriber. Back it up, move it, delete it. Your call.

Direct API calls

Audio uploads go from your Mac straight to OpenAI and AssemblyAI using your keys. No relay, no proxy, no server-side storage on our end.

Zero telemetry

No crash reporting, no usage analytics, no auto-update phone-home. The only network traffic is to the AI providers you configure.

Troubleshooting

If something doesn’t work on first launch.

Common issues and quick fixes. If you hit something not covered here, open a GitHub issue.

macOS says the app “cannot be opened because Apple cannot check it for malicious software”
That’s Gatekeeper blocking the unsigned build. Right-click (or Control-click) the app in Applications and choose Open, then click Open on the warning dialog. After that first time, double-click works normally. We’ll ship signed + notarized builds once Apple Developer enrollment completes.
The .dmg won’t mount when I double-click it
Strip the quarantine flag your browser set on download, then mount from Terminal:
$ xattr -d com.apple.quarantine ~/Downloads/Videoscriber-*.dmg $ open ~/Downloads/Videoscriber-*.dmg
Or re-download with curl -L -O <release-url>, which skips the quarantine attribute entirely.
“Python not found” on first launch
Install Python 3.10 or newer. Easiest: brew install python@3.12. After install, run which python3 in Terminal to confirm. Then re-launch Videoscriber.
“ffmpeg not found” on first launch
brew install ffmpeg. Videoscriber uses it to extract audio and enhance difficult tracks before transcription. Without it, uploads fail immediately.
First launch hangs for a minute with no window
That’s expected. Videoscriber is creating a Python virtual environment and installing dependencies at ~/Library/Application Support/Videoscriber/venv. It only happens once. If it takes longer than ~3 minutes, quit the app and run tail -f ~/Library/Logs/Videoscriber/*.log in Terminal while you relaunch.
Transcription fails with an authentication error
Your API key is missing, expired, or has no billing attached. Go to Settings in the app and paste fresh keys. For AssemblyAI, check assemblyai.com/dashboard; for OpenAI, check platform.openai.com/api-keys and confirm you have a positive balance.
How do I update to a newer release?
Download the latest .dmg from this page and drag it into Applications, replacing the old copy. Your data, API keys, and venv are stored outside the app bundle, so they carry over. Auto-update is on the roadmap.
Is there a Windows or Linux build?
Not yet. The Electron app is Mac-only for now. The underlying FastAPI backend runs anywhere Python + ffmpeg do, so you can git clone the repo and run it on Windows or Linux directly. A packaged Windows build is on the roadmap.
Where is my data stored?
~/Library/Application Support/Videoscriber/: database (data/), uploaded recordings (uploads/), extracted audio (audio/), API keys (.env), and the Python venv. To wipe everything and start fresh: quit the app, delete that folder, relaunch.
